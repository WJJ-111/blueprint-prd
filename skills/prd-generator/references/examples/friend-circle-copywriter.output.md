# PRD: 朋友圈文案助手

## 1. 产品概述
- **产品名称**：朋友圈文案助手
- **核心问题**：帮助运营人员快速生成高质量朋友圈文案，降低内容创作门槛。
- **目标用户**：需要通过朋友圈进行产品推广、品牌运营或个人 IP 打造的从业者，包括微商、销售、自媒体从业者和小企业主。
- **平台**：网页应用（Web App）

## 2. 功能需求

### MVP 功能（v1）

| 功能名称 | 功能描述 | 用户价值 |
|---------|---------|---------|
| 场景分类选择 | 提供 7 个预设场景标签供用户单选，帮助用户快速确定文案方向。 | 降低输入门槛，提升生成效率。 |
| 场景自由描述 | 提供最多 200 字的场景描述输入框，用于补充具体需求。 | 支持更细粒度的个性化生成。 |
| 场景校验逻辑 | 场景分类和场景描述至少填写一项，未满足时提示用户补充。 | 保证模型具备必要上下文。 |
| 风格选择 | 提供 6 个预设风格选项供用户单选。 | 控制输出文案的表达调性。 |
| 字数范围选择 | 提供 4 档字数范围供用户单选。 | 适配不同内容长度需求。 |
| 参考示例输入 | 支持填写最多 3 条参考文案，每条最多 500 字。 | 提高生成内容与用户偏好的贴合度。 |
| 一次生成 3 条文案 | 基于当前输入条件一次返回 3 条不同文案。 | 提供多样选择，提高满意度。 |
| 一键复制 | 每条文案结果支持复制，并展示复制成功提示。 | 缩短从生成到发布的操作路径。 |
| 换一批 | 在保留原条件的前提下重新生成 3 条文案。 | 快速获取替代结果。 |
| 返回修改条件 | 用户可返回输入页修改参数，并保留原已填内容。 | 便于迭代调整输入条件。 |

### 未来规划（v2+）
- 历史记录
- 收藏功能
- 文案模板库
- 批量生成
- 配图建议
- 多语言支持

## 3. 页面设计

### 页面结构
- **输入页**：包含场景分类选择、场景描述输入、风格选择、字数范围选择、参考示例输入和生成按钮。
- **结果页**：展示 3 条文案结果卡片，并提供复制、换一批和修改条件按钮。
- **交互流程**：采用单页应用形式，通过输入页和结果页两个视图切换完成主流程。

### 视觉风格
- 采用科技感暗色主题，使用深色背景和高亮强调色。
- 保证结果卡片层级清晰、文案可读性高。
- 生成过程中展示明确的 loading 状态。

## 4. 技术规格

| 项目 | 要求 |
|-----|------|
| 平台 | 网页应用（Web App） |
| 用户认证 | 不需要 |
| 数据存储 | 不需要 |
| 技术栈建议 | 前端可采用 React 或 Vue；后端可采用 Serverless Function 中转模型调用；部署可采用 Vercel 或 Netlify 等平台。 |

## 5. 非功能需求
- 性能要求：页面首次加载时间不超过 2 秒；AI 文案生成响应时间不超过 10 秒。
- 安全要求：API Key 仅存储在服务端环境变量中；前端做输入长度限制与基础 XSS 防护；增加简单频率限制。
- 可用性要求：表单校验需提供实时反馈；生成失败时保留用户已填写内容；移动端点击区域不小于 44px；兼容主流浏览器。

## 6. 项目范围

**在范围内：**
- 单页 Web 应用开发
- 输入页与结果页视图切换
- 场景分类与自由描述输入
- 风格和字数范围选择
- 参考示例输入
- 生成 3 条文案
- 复制、换一批、返回修改条件
- 响应式适配
- Serverless Function 中转模型调用

**不在范围内（本期）：**
- 用户登录注册
- 数据库存储与历史记录
- 收藏功能
- 模板库
- 批量生成
- 配图建议
- 多语言支持
- 后台管理
- 原生 App 与小程序
